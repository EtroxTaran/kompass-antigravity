## Writing Guide Standards

### Guide Structure Template

```markdown
# Guide Title

**Purpose**: Clear statement of what this guide accomplishes
**Audience**: Who should use this guide
**Prerequisites**: What you need before starting
**Estimated Time**: How long this should take

## Quick Summary

[Brief overview of key points]

## Step-by-Step Instructions

[Detailed procedures]

## Troubleshooting

[Common issues and solutions]

## Related Resources

[Links to related documentation]
```

### Quality Standards

- ‚úÖ **Clear prerequisites** - State what users need before starting
- ‚úÖ **Step-by-step format** - Break complex procedures into manageable steps
- ‚úÖ **Code examples** - Include working examples for technical procedures
- ‚úÖ **Screenshots/visuals** - Add visuals where helpful
- ‚úÖ **Troubleshooting section** - Address common issues
- ‚úÖ **Testing instructions** - How to verify the guide worked
- ‚úÖ **Update dates** - Keep modification dates current

## Maintenance

### Update Schedule

- **Major releases**: All guides reviewed and updated
- **Feature additions**: Affected guides updated within 1 week
- **Bug fixes**: Troubleshooting sections updated as needed
- **Quarterly review**: Check all guides for accuracy and relevance

### Contribution Process

1. Identify need for new guide or update
2. Create Linear issue with guide scope
3. Draft guide using template structure
4. Review with relevant team (dev/product/UX)
5. Test guide with fresh environment/user
6. Update cross-references and navigation
7. Publish and announce to team

### Guide Ownership

- **Development Guides**: Engineering team
- **User Guides**: Product team with UX review
- **Implementation Guides**: Feature development team
- **Getting Started**: Shared ownership (Engineering + Product)
